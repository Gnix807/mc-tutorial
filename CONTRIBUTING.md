# 贡献指南

感谢你对这份教程的关注！以下是参与贡献的详细流程和规范。

## 贡献方式

### 报告问题

如果你发现教程中的错误、过时信息或有改进建议，请提交 [Issue](https://github.com/你的用户名/仓库名/issues)，描述清楚问题和位置（章节 + 段落）。

### 提交内容

1. **Fork** 本仓库
2. 创建你的分支：`git checkout -b feature/你的改动描述`
3. 做出修改并提交
4. 推送分支：`git push origin feature/你的改动描述`
5. 提交 **Pull Request**

## 写作规范

### 文件命名

- 章节目录：`chXX-英文简写/`，如 `ch01-getting-started/`
- 节文件：`XX-英文简写.md`，如 `01-buy-and-install.md`
- 图片统一放 `static/images/chXX/` 下，按编号命名

### 内容格式

所有内容使用 Markdown 编写。教程使用了自定义 Hugo 短代码来增强排版，以下是常用语法：

#### 提示框

```markdown
> [!NOTE]
> 这是一条注意事项。

> [!TIP]
> 这是一个小技巧。

> [!WARNING]
> 这是一个警告。
```

#### 图片

```markdown
{{< image src="/images/ch02/1.png" alt="描述" title="标题" width="750" >}}
```

#### 卡片列表

```markdown
{{< card-list columns="2" >}}
- **标题一**
  描述内容
- **标题二**
  描述内容
{{< /card-list >}}
```

#### 时间线步骤

```markdown
{{< timeline >}}
{{< timeline-item "步骤 1" >}}
内容
{{< /timeline-item >}}
{{< /timeline >}}
```

#### 行内标签

```markdown
{{< badge text="推荐" >}}
{{< key >}}Ctrl + S{{< /key >}}
{{< tip "提示文字" >}}悬停查看{{< /tip >}}
```

### 引号规范

正文中的双引号统一使用直角引号「」，短代码参数中的引号使用英文双引号 "。

### 写作风格

请参考已完成的第一章和 2.1 节：

- 亲切但不轻浮——像朋友在教你，而不是教科书在说教
- 专业但不枯燥——保留术语但配以直白解释
- 幽默收在分寸——偶尔调侃，不喧宾夺主
- 面向纯萌新——不要假设读者已经知道任何 Minecraft 知识

## 截图规范

- 图片格式：PNG 或 WebP
- 分辨率：宽度建议 1920×1080，文章中统一使用 `width="750"`
- 截取完整的游戏窗口，不包括桌面任务栏
- 如果使用 Minecraft Wiki 或其他来源的图片，需在附录中注明出处

## 数据准确性

本教程基于 **Minecraft Java 版 26.2** 编写。撰写内容时请以 [Minecraft Wiki](https://zh.minecraft.wiki/) 为权威参考，确保数据（合成配方、矿物分布高度、生物属性等）与游戏实际一致。

## 提交规范

- 一次提交尽量只做一件事
- 提交信息使用中文，以动词开头，末尾不加标点
- 中文与英文、数字之间留一个空格

## 本地开发

```bash
# 安装 Hugo（需 v0.158+）
winget install Hugo.Hugo.Extended

# 初始化子模块
git submodule update --init

# 启动开发服务器（含草稿）
hugo server --bind 0.0.0.0 --port 1313 --buildDrafts

# 构建正式版本
hugo --gc --cleanDestinationDir
```
